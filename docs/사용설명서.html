BODA.CMS 간단 사용설명서 현장용 · 로봇 건강 지킴이

BODA.CMS는 협동로봇의 상태를 로봇 작업을 전혀 방해하지 않고 지켜보다가, 평소와 다른 점이 생기면 고장 나기 전에 미리 알려주는 프로그램입니다. 로봇에 아무 프로그램도 설치하지 않으며, 데이터를 읽기만 합니다.

1. 시작하기 — 3분이면 됩니다

1프로그램 실행
바탕화면의 BODA.CMS를 실행합니다.
2제조사 선택
맨 위 제조사 목록에서 우리 로봇의 제조사(두산, JAKA 등)를 고릅니다.
💡 로봇 없이 연습하고 싶으면 "시뮬레이터 (가상 데이터)"를 고르세요 — 가짜 로봇이 움직이며 모든 기능을 체험할 수 있습니다.
3IP 주소 입력
로봇 컨트롤러의 IP 주소를 입력합니다. (모르면 설비 담당자에게 문의 — 두산 기본값은 192.168.137.100)
4모니터링 시작
카드마다 있는 모니터링 시작 버튼을 누릅니다. 상태등이 초록으로 바뀌고 숫자가 움직이면 성공입니다.
5이제 그냥 두면 됩니다
약 1분간 로봇의 "평소 상태"를 스스로 학습한 뒤(카드에 CBM 학습 % 표시), 자동으로 감시가 시작됩니다. 이상이 생기면 화면 아래 알림에 나타납니다.
⚠️ 이 프로그램은 보기만 합니다. 로봇을 움직이거나 멈추게 할 수 없으므로 켜 둔 채로 생산을 계속해도 안전합니다.

2. 화면 읽는 법

카드 한 장 = 데이터 연결선 하나

로봇 한 대에서 데이터를 받아오는 길이 보통 두 개 있습니다(예: 두산 Modbus / 두산 DRFL). 길마다 카드가 한 장씩 생기며, 카드 제목 옆 괄호는 다음 뜻입니다:

표기뜻
Basic / Pro이 연결선으로 받을 수 있는 데이터의 상세함 등급. Basic = 기본(위치 중심), Pro = 상세(관절별 힘·전류까지). 프로그램이 자동으로 판별합니다.
10Hz / 100Hz1초에 데이터를 받아오는 횟수.
패시브"읽기 전용" — 로봇 작업을 방해하지 않는다는 표시입니다.

상태등 색깔

색뜻조치
회색대기 중모니터링 시작을 누르세요
주황연결하는 중잠시 기다리세요
초록정상 수신 중-
빨강연결이 끊김/실패8장 문제 해결 참조

숫자 표와 체크박스

검은 판에 관절(J1~J6)별 숫자가 실시간으로 표시됩니다. 표 위의 체크박스로 보고 싶은 항목만 골라 볼 수 있습니다. ~_raw가 붙은 항목은 아직 단위 환산 전의 "원시 숫자"라서 값 자체보다 변화 추세를 보는 용도입니다.

💡 화면 곳곳의 물음표(?) 아이콘을 누르면 해당 항목의 자세한 설명이 바로 열립니다.

3. 건강 점수 이해하기 (화면의 "CBM")

쉽게 말해 "로봇의 평소 몸 상태를 외워뒀다가, 평소와 다르면 알려주는 기능"입니다. 화면에는 CBM이라고 표시됩니다.

1모니터링을 시작하면 약 1분간 평소 상태를 학습합니다. (카드에 "CBM 학습 %" 표시)
⚠️ 학습하는 1분 동안은 로봇이 정상적으로 작업 중이어야 합니다. 고장 난 상태로 학습하면 그 상태를 "평소"로 외워버립니다.
2학습이 끝나면 건강 점수(0~100점)가 표시됩니다:

80~100점 평소와 같음 — 안심   50~79점 평소와 조금 다름 — 지켜보기   0~49점 평소와 많이 다름 — 알림 확인!
3점수가 낮아지면 어느 관절의 어떤 값이 문제인지 함께 표시됩니다. 예: J3 전류A z=4.2 = "3번 관절의 전류가 평소 범위를 4.2배 벗어남".

4. AI 이상 감지 이해하기 (화면의 "ML")

건강 점수가 "값이 평소보다 큰가/작은가"를 본다면, AI 이상 감지는 "값이 움직이는 모양새가 평소와 다른가"를 봅니다 — 떨림, 출렁임, 계단처럼 튀는 패턴 등 사람이 숫자만 봐서는 놓치기 쉬운 변화를 잡아냅니다. 화면에는 ML로 표시됩니다.

화면 표시뜻
ML 정상움직임 패턴이 평소와 같음
ML 이상 n건평소와 다른 패턴 감지 — 알림에서 상세 확인
ML 모델 없음AI 기능이 설치되지 않음(프로그램 폴더에 models 폴더 필요) — 건강 점수만으로도 감시는 됩니다

5. 알림이 뜨면 이렇게 하세요

화면 아래 "CBM 알림" 칸에 최신순으로 쌓입니다. 색과 종류별 대처:

알림 종류쉬운 뜻권장 조치
급변 (빨강) 값이 갑자기 크게 변함 충돌·간섭·부하 급변 의심. 로봇과 주변을 즉시 확인하고, 반복되면 보전 담당자에게 알리세요.
드리프트 (주황) 값이 서서히 변하는 중 마모·발열 같은 점진적 변화 의심. 당장 세울 일은 아니지만 기록해 두고 추세를 지켜보세요.
ML 이상 (주황) AI가 평소와 다른 움직임 패턴을 감지 해당 관절의 차트를 열어 파형을 확인하세요. 급변/드리프트와 같이 뜨면 신뢰도가 높습니다.
복귀 (초록) 문제가 있던 값이 정상으로 돌아옴 일시적 현상이었을 수 있습니다. 자주 반복되면 그것 자체가 신호이니 기록하세요.

알림 문구의 z=4.2는 "평소 흔들리는 폭의 4.2배만큼 벗어났다"는 뜻입니다. 숫자가 클수록 확실한 이상입니다.

6. 차트 보는 법

1카드의 차트 버튼을 누르면 숫자 표가 실시간 그래프(최근 30초)로 바뀝니다.
2차트 신호 목록에서 보고 싶은 항목 하나를 고릅니다. 관절 6개(J1~J6)가 색깔별 선으로 그려집니다.
3알림이 가리킨 관절의 선이 다른 선과 다르게 움직이는지 확인하세요. 다시 차트를 누르면 표로 돌아옵니다.

7. 사무실에서 원격으로 보기 (웹 대시보드)

수집 서버(BODA.CMS.Collector)가 돌고 있다면, 같은 네트워크의 아무 PC에서나 브라우저로 볼 수 있습니다 — 설치 불필요:

http://수집서버주소:5100

로봇별 카드(연결 상태·수신 속도·건강 점수·AI 판정)와 알림이 1초마다 갱신됩니다. 수집 서버는 사람이 없어도 24시간 감시와 기록을 계속합니다.

8. 문제가 생겼을 때

증상확인할 것
"연결 실패 / 연결 타임아웃" ① 랜선·IP 주소 확인 ② 로봇 컨트롤러 전원 확인 ③ 같은 네트워크인지 확인(PC에서 ping 로봇IP)
"연결 끊김" (빨강) 네트워크가 잠시 끊긴 것 — 원인 해결 후 모니터링 시작을 다시 누르세요. (수집 서버는 자동으로 재접속합니다)
"라이선스 등급 초과" 현재 라이선스로 쓸 수 없는 상세(Pro) 채널입니다 — 구매처에 등급 업그레이드를 문의하세요.
건강 점수가 자꾸 낮게 나옴 학습 중에 로봇이 멈춰 있었거나 평소와 다른 작업을 했을 수 있습니다 — 정상 작업 중에 모니터링을 다시 시작해 재학습하세요.
숫자가 안 보이고 "(모니터링 시작 전)" 카드의 모니터링 시작 버튼을 눌렀는지 확인하세요.
프로그램 오류 창이 뜸 프로그램은 계속 동작합니다. 오류 기록이 %TEMP%\BODA.CMS.crash.log에 남으니 담당자에게 전달해 주세요.

9. 용어 사전 — 화면 표기 ↔ 쉬운 말

화면 표기쉬운 말설명
CBM건강 상태 감시평소 상태를 외워뒀다가 달라지면 알려주는 기능 (Condition Based Monitoring)
MLAI 이상 감지값의 "움직임 패턴"이 평소와 다른지 AI가 판단 (Machine Learning)
기준선평소 상태정상 작업 1분간 학습한 평균적인 값의 범위
z (예: z=4.2)벗어난 정도평소 흔들림 폭의 몇 배나 벗어났는지 — 클수록 확실한 이상
급변갑자기 변함충돌·간섭처럼 순간적으로 값이 튐
드리프트서서히 변함마모·발열처럼 천천히 조금씩 변함
패시브읽기 전용로봇 작업을 절대 방해하지 않는 연결 방식
채널데이터 연결선로봇에서 데이터를 받아오는 통로 (한 로봇에 여러 개 가능)
Basic / Pro기본 / 상세 등급연결선으로 받을 수 있는 데이터의 상세함 — 자동 판별
Hz초당 횟수10Hz = 1초에 10번 데이터 수신
토크(Nm)관절이 내는 힘무리하면 커짐 — 마모·부하 이상의 핵심 지표
~_raw원시 숫자단위 환산 전의 값 — 크기보다 변화 추세를 보는 용도
J1~J61~6번 관절로봇 바닥(J1)부터 손목(J6)까지의 회전축

10. 관리자용 — 감시 서버 설치·운영 (설비/IT 담당자)

프로그램은 두 가지입니다. 역할이 다릅니다:

프로그램역할어디에
BODA.CMS (모니터 화면)사람이 보면서 진단하는 화면 프로그램현장/사무실 PC — 필요할 때 실행
BODA.CMS.Collector (감시 서버)사람이 없어도 24시간 데이터 수집·기록·이상 감시 + 웹 대시보드 제공공장 내 상시 가동 PC 1대 — Windows 서비스로 등록

설치 순서 (감시 서버)

1설치 파일 준비
공급사에서 받은 BODA.CMS-collector-버전-win-x64.zip을 고정 폴더(예: C:\BODA\collector)에 풉니다. 💡 .NET 등 별도 프로그램 설치가 필요 없습니다 — 압축만 풀면 됩니다.
2감시할 로봇 등록
푼 폴더의 appsettings.json을 메모장으로 열어 Robots 부분에 로봇을 적습니다: 
"Robots": [
  { "RobotId": "1라인-두산-01", "Vendor": "doosan", "Host": "192.168.1.100" },
  { "RobotId": "2라인-두산-02", "Vendor": "doosan", "Host": "192.168.1.101" }
]
RobotId는 대시보드에 표시될 이름(자유), Vendor는 제조사 코드(doosan / jaka / sim), Host는 로봇 컨트롤러 IP입니다. 여러 대를 쉼표로 이어 적으면 동시에 감시합니다.
3라이선스 배치 (구매한 경우)
받은 license.json 파일을 exe 옆에 복사합니다. 없으면 평가판으로 동작하며, 파일이 잘못되거나 기한이 지나면 기본(Basic) 기능만 동작합니다.
4Windows 서비스로 등록
관리자 권한 PowerShell에서: 
powershell -File tools\install-service.ps1 -ExePath "C:\BODA\collector\BODA.CMS.Collector.exe"
등록되면 즉시 시작되고, 이후 PC를 재부팅해도 자동으로 다시 시작됩니다. 프로그램이 비정상 종료돼도 스스로 재시작합니다(10초→30초→60초 간격).
5동작 확인
같은 PC 브라우저에서 http://localhost:5100 — 로봇 카드가 보이면 성공. 서비스 상태는 services.msc에서 "BODA.CMS.Collector"가 실행 중인지 확인.

다른 PC에서 대시보드 열기

기본 설정은 보안을 위해 그 PC 안에서만 열립니다. 사무실 등 다른 PC에서 보려면:

1appsettings.json 맨 위를 "Urls": "http://0.0.0.0:5100"으로 수정 후 서비스 재시작.
2관리자 PowerShell에서 방화벽 허용: 
netsh advfirewall firewall add rule name="BODA.CMS Dashboard" dir=in action=allow protocol=TCP localport=5100
⚠️ 대시보드에는 아직 로그인이 없습니다 — 공장 내부망까지만 여세요. 인터넷에 직접 노출 금지.

일상 운영

하고 싶은 일방법
설정 변경 반영appsettings.json 수정 → services.msc에서 서비스 다시 시작
동작 기록(로그) 보기Windows 이벤트 뷰어 → Windows 로그 → 응용 프로그램 → 원본 "BODA.CMS.Collector"
프로그램 업데이트서비스 중지 → 폴더의 파일을 새 버전으로 교체(설정 파일은 유지) → 서비스 시작
서비스 제거관리자 PowerShell: powershell -File tools\install-service.ps1 -Uninstall
데이터 저장(DB) 켜기appsettings.json의 Storage:Enabled를 true로 + DB 접속 문자열 설정 (TimescaleDB 필요 — 공급사 문의)

AI 모델 재학습 — 우리 로봇에 맞게 똑똑하게 만들기

처음 설치된 AI 모델은 범용 모델이라 우리 로봇의 평소 움직임을 모릅니다. 그래서 실제로는 멀쩡한데 "ML 이상"이 자꾸 뜰 수 있습니다. 데이터 저장(DB)을 켜고 정상 운전 데이터가 하루 이상 쌓이면, 그 데이터로 재학습해 우리 로봇 전용 모델로 교체할 수 있습니다. 오탐이 크게 줄어듭니다.

1재학습 실행 (파이썬이 설치된 개발/관리 PC의 저장소 폴더에서)
